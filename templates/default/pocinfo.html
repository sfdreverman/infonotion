This prototype shows:

Low-code architecture
The application structure is intended for low-code.
Out-of-the-box, you can use the website to create datatypes and data.
This means the application is data-independent! And it needs to stay that way in order to keep it low-code.
The intention is to keep the code as data-independent as possible. (Of course, the exception are the meta-data that enable this low-code architecture)

With add-ons, you can add your own functionality to it.
When well-written, the add-ons fit in the low-code architecture. How?
The code will contain:
- meta-level handling of functionality.
- the meta-types (and -data) that support this functionality


Views
This addon delivers functionality to view data. It enables you to create custom queries and views for any datatype.
Yes, it's rudimentary. A 'query builder' is still needed to abstract the neo4j-query part.
